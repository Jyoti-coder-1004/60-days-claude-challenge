# Day 37: Build Task Compass (Tech Company Edition)

## 📋 Project Overview
**Task Compass** is an interactive organizational simulation designed to teach how work flows through a modern **Tech Company**. Instead of treating roles like static job titles, this simulator explores core management concepts: ownership, delegation, collaboration, escalation, and cross-functional workflows.

### 🚀 Key Learnings & Core Concepts
- **True Ownership vs. Support:** Learned how to distinguish between the primary driver of a task (e.g., QA Engineer for regression testing) and the cross-functional partners who assist them.
- **Workflow Routing Pipelines:** Mapped out sequential dependencies showing how a simple customer bug moves systematically from Support ➔ QA ➔ Engineering ➔ Product Management ➔ End User.
- **Cross-Functional Collaboration:** Simulated high-stakes scenarios (like sudden drops in user retention or massive API outages) requiring the alignment of up to 4 distinct departments simultaneously.
- **Organizational Realism:** Shifted away from an "exam/quiz" mindset toward an interactive strategy-game style that accurately reflects real-world engineering and product operations.

---

## 🛠️ Tech Stack & Architecture
- **Frontend:** Semantic HTML5, CSS3 Custom Variables (Modern Glassmorphism Design).
- **Interactions:** Vanilla JavaScript Native Drag-and-Drop API.
- **State Management:** In-memory decoupled configuration object tracking progress across 3 functional stages.
- **Visuals:** Pure CSS-rendered dynamic performance radar metric dashboard.

## 📊 Self-Reflection & Insight Dashboard
- **Strength Identified:** Strong structural understanding of linear task routing and technical bug assignment pipelines.
- **Area for Optimization:** Initially tended to over-assign engineering roles to tasks that were primarily product strategy or documentation updates.
- **Core Organizational Insight:** Most critical high-impact issues in a tech ecosystem are never solved in silos; systemic health relies entirely on clear communication lanes between engineering, support, and product.
